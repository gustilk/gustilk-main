# Gûstîlk - Dating App for Yezidies

A specialized dating platform designed specifically for the Yezidi community, respecting cultural traditions and the caste system.

## Features

### Core Features
- **Caste-Based Matching**: Sheikh, Pir, and Mirid users can only see and match with their own caste
- **Multi-Country Support**: USA, Canada, Australia, Germany, Holland, Sweden, Belgium, Turkey, Iraq, Armenia, Georgia, Russia, France
- **Location-Based Services**: Automatic location detection
- **Premium Subscription**: $5/month (Free for Iraq users based on location)
- **Multi-Language Support**: All languages for supported countries with user-selectable preferences
- **Live Video**: Real-time video chat functionality
- **Events**: Community event features
- **Verification System**: Mandatory selfie verification for all users
- **Reporting System**: Users can report misrepresentation and bad behavior

### Premium Features
- View messages
- See matches
- See who liked you
- View profile visitors

### Security & Community
- Mandatory profile pictures (cannot sign up without photos)
- Face verification via selfie
- Community guidelines based on Yezidi faith
- User agreement upon signup
- Admin tools for moderation
- Account suspension/deletion for violations

### Payment Integration
- Country-specific payment methods (most popular per region)
- Free premium for Iraq (location-detected)
- Stripe integration for international payments

## Setup Instructions

### Prerequisites
- Node.js 16+ and npm
- Firebase account and project
- Stripe account (for payments)

### 1. Clone and Install

```bash
# Install root dependencies
npm install

# Install client dependencies
cd client
npm install

# Install server dependencies
cd ../server
npm install
```

### 2. Environment Configuration

Create `.env` file in the root directory (use `.env.example` as template):

```env
# Firebase Configuration
FIREBASE_API_KEY=your_firebase_api_key
FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
FIREBASE_PROJECT_ID=your_project_id
FIREBASE_STORAGE_BUCKET=your_project.appspot.com
FIREBASE_MESSAGING_SENDER_ID=your_sender_id
FIREBASE_APP_ID=your_app_id

# Server Configuration
PORT=5000
NODE_ENV=development
JWT_SECRET=your_jwt_secret_key

# Stripe Configuration
STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key

# Payment Method IDs by Country
STRIPE_PAYMENT_METHOD_US=pm_card
STRIPE_PAYMENT_METHOD_CA=pm_card
STRIPE_PAYMENT_METHOD_DE=pm_sepa_debit
# ... add more as needed

# Admin Configuration
ADMIN_EMAIL=admin@gustilk.com
```

### 3. Firebase Setup

1. Create a Firebase project at https://console.firebase.google.com
2. Enable Authentication (Email/Password)
3. Create Firestore Database
4. Set up Storage for profile pictures
5. Download service account key for Firebase Admin SDK
6. Place it in `server/config/serviceAccountKey.json`

### 4. Stripe Setup

1. Create Stripe account
2. Get API keys from dashboard
3. Configure payment methods for each country
4. Set up webhooks for subscription events

### 5. Database Schema (Firestore)

```
users/
  {userId}/
    - email
    - caste (sheikh|pir|mirid)
    - country
    - location {lat, lng}
    - isPremium
    - premiumExpiryDate
    - photos []
    - isVerified
    - verificationPhoto
    - language
    - createdAt
    - lastActive

matches/
  {matchId}/
    - users [userId1, userId2]
    - status (pending|matched|rejected)
    - createdAt

messages/
  {conversationId}/
    messages/
      {messageId}/
        - senderId
        - text
        - timestamp
        - read

reports/
  {reportId}/
    - reporterId
    - reportedUserId
    - reason
    - description
    - status
    - createdAt

events/
  {eventId}/
    - title
    - description
    - date
    - location
    - country
    - caste
    - createdBy
```

## Running the Application

### Development Mode

```bash
# Terminal 1 - Run backend
cd server
npm run dev

# Terminal 2 - Run frontend
cd client
npm start
```

The app will be available at:
- Frontend: http://localhost:3000
- Backend API: http://localhost:5000

### Production Build

```bash
# Build frontend
cd client
npm run build

# Start production server
cd ../server
npm start
```

## Deployment

### Frontend (Vercel/Netlify)
1. Build the client: `cd client && npm run build`
2. Deploy the `build` folder
3. Set environment variables in hosting platform

### Backend (Heroku/DigitalOcean)
1. Push server code to hosting platform
2. Set environment variables
3. Ensure PORT is configurable
4. Run `npm start`
